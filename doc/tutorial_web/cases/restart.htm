Run WPS

No need to run WPS again.
 

Run WRF

  • Make sure you are in the WRFV2/test/em_real directory
     
     
  • Due to time and space limitations, it is best to test this with a single domain.
     
     
  • First we need to run the model as before, but this time we are going to generate restart files. To do this, edit the namelist (shown below is the JAN00 case, but it will work the same for any case you are running)
         run_hours = 1,
         start_year = 2000,
         start_month = 01
         start_day = 24,
         start_hour = 12,
         end_year = 2000,
         end_month = 01,
         end_day = 25,
         end_hour = 12,
         input_from_file = .true.,
         history_interval = 30,
         restart = .false.,
         restart_interval = 60,
     
    Set run_hours = 1 (this is NOT needed for a restart, but is used in this example to run for a short time)
    Set restart_interval = 60 (This will create a wrfrst* file every 60 minutes)
     
  • No need to run real.exe again, so just run wrf.exe
     
     
  • Second, once the initial run is complete, we will restart the model and run for a second hour. To do this edit the namelist again
         run_hours = 1,
         start_year = 2000,
         start_month = 01
         start_day = 24,
         start_hour = 13,
         end_year = 2000,
         end_month = 01,
         end_day = 25,
         end_hour = 12,
         input_from_file = .true.,
         history_interval = 30,
         restart = .true.,
         restart_interval = 60,
     
    Set run_hours = 1 (this is NOT needed for a restart, we simpily set this in this example to show how to run for another hour)
    Set start_hour = 13 (in the initial run, we run from 12 to 13Z, so now we will start from 13Z)
    Set restart = .true. (indicating that this is a restart run)
     
  • No need to run real.exe again, so just run wrf.exe
     
     

If this was successful, you can continue to run another case of your choice, or generate graphics for your case.
